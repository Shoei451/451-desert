# Ideas Manager App - セットアップガイド

アイデアを記録・整理するための個人用アプリです。Supabaseを使用して、複数のデバイスから安全にアクセスできます。

## 機能

- **アイデア名**: 各アイデアにタイトルをつけて管理
- **タグ**: カンマ区切りで複数のタグを設定可能
- **詳細メモ**: アイデアの詳細を自由に記述
- **タイムスタンプ**: 作成日時と最終更新日時を自動記録
- **フィルター & ソート**: タグでフィルタリング、日時や名前でソート
- **統計表示**: 総数、タグ数、今週の新規アイデア数を表示
- **エクスポート/インポート**: JSONファイルでデータをバックアップ・復元

## セットアップ手順

### 1. Supabaseデータベースのセットアップ

1. [Supabase](https://supabase.com)のダッシュボードにログイン
2. 左側のメニューから「SQL Editor」を選択
3. `supabase-ideas-setup.sql`ファイルの内容をコピー
4. SQLエディタに貼り付けて「Run」をクリック

これにより以下が作成されます：
- `ideas_app`テーブル
- パフォーマンス向上のためのインデックス
- ユーザーデータ保護のためのRLSポリシー

### 2. ファイルの配置

プロジェクト構造：
```
your-project/
├── ideas-app.html              # メインアプリファイル
├── index-with-ideas.html       # 更新されたホームページ（ideasアプリへのリンク付き）
├── css/
│   └── solar-ideas-style.css   # アプリのスタイルシート
├── images/
│   └── favicon-solar.png       # ファビコン（既存）
└── supabase-ideas-setup.sql    # データベースセットアップスクリプト
```

### 3. 既存のindex.htmlの更新（オプション）

既存の`index.html`にIdeas Managerへのリンクを追加したい場合：

`index-with-ideas.html`の内容で既存の`index.html`を置き換えるか、
または以下のカードを`<section class="cards-container">`内に追加：

```html
<a href="ideas-app.html" class="card">
    <div class="card-icon">💡</div>
    <h3>Ideas Manager</h3>
    <p>アイデアを記録・整理・発展させる思考整理ツール</p>
</a>
```

## 使い方

### 基本操作

1. **新しいアイデアを追加**
   - 「＋ 新しいアイデア」ボタンをクリック
   - アイデア名とタグを入力
   - 詳細メモを記入（オプション）
   - 「保存」をクリック

2. **アイデアを編集**
   - アイデアカードをクリック
   - 内容を編集して「保存」

3. **タグでフィルター**
   - 右上のドロップダウンからタグを選択
   - タグクラウドのタグをクリック

4. **ソート順変更**
   - ドロップダウンから「更新日時順」「作成日時順」「名前順」を選択

### データ管理

- **エクスポート**: 「Export JSON」でバックアップを作成
- **インポート**: 「Import JSON」でバックアップから復元
- **全消去**: 「全消去」ボタン（注意: 元に戻せません）

## 技術仕様

### データ構造

```javascript
{
  id: string,           // 一意のID
  user_id: uuid,        // ユーザーID（Supabase Auth）
  name: string,         // アイデア名
  tags: string,         // タグ（カンマ区切り）
  content: string,      // 詳細メモ
  created_at: timestamp,   // 作成日時
  modified_at: timestamp   // 最終更新日時
}
```

### セキュリティ

- Row Level Security (RLS)により、ユーザーは自分のデータのみアクセス可能
- 認証はSupabase Authで管理
- すべてのデータベース操作はサーバー側で検証

### リアルタイム同期

- Supabaseのリアルタイム機能により、複数デバイス間で自動同期
- 他のデバイスでの変更が即座に反映

## トラブルシューティング

### データが表示されない

1. Supabaseでテーブルが正しく作成されているか確認
2. RLSポリシーが有効になっているか確認
3. ブラウザのコンソールでエラーメッセージを確認

### 同期エラー

1. インターネット接続を確認
2. Supabaseのステータスページを確認
3. 認証トークンの有効期限が切れていないか確認（再ログインしてください）

### エクスポート/インポートの問題

- JSONファイルの形式が正しいか確認
- ファイルサイズが大きすぎないか確認（ブラウザの制限）

## カスタマイズ
